# HylianScan v0.9 - Release Summary

Date: 2026-06-14

## Summary

Version `v0.9` focuses on reliability, automation readiness, terminal portability, and maintainable service probing. The scanner keeps the v0.8 TCP and passive discovery behavior while adding a stronger testing foundation and cleaner internal extension points.

## Key Changes

- Marked the project release version as `v0.9`.
- Added automation-friendly quiet mode for plain terminal output without ANSI styling or decorative panels.
- Added a protocol probe registry for HTTP, HTTPS, SMTP, SMTPS, FTP, FTPS, generic TLS metadata ports, and unknown fallback ports.
- Expanded JSON intelligence with structured HTTP metadata and TLS risk indicators.
- Improved Windows-safe terminal fallbacks while preserving Linux/Kali terminal behavior.
- Added phase-oriented TCP live output and an RGB progress bar for normal interactive scans.
- Refined the final TCP report into a structured `PORT / STATE / SERVICE / VERSION` layout.
- Added common alternate HTTP and HTTPS port mappings for service naming, probing, and URL hints.
- Added optional TCP max-rate pacing to control how quickly new connection attempts are started.
- Added SMTP STARTTLS upgrade probing so plain SMTP services can expose TLS metadata when supported.
- Added structured TCP probe metadata to JSON findings, including probe name, method, transport security mode, and STARTTLS status.
- Added structured HTTP Set-Cookie parsing with cookie attributes, prefixes, and security observations in TCP JSON reports.
- Added structured HTTP security-header observations for common response headers in TCP JSON reports.
- Added standard-library tests for CLI helpers, port helpers, banner probing, TLS analysis, JSON export, output helpers, quiet mode, and localhost mock services.

## Testing Scope

The v0.9 reliability phase intentionally prioritizes pure logic and safe localhost integration tests:

- CLI argument normalization and mode validation.
- TCP port normalization and service-name helpers.
- Protocol-aware banner grabbing dispatch through the probe registry.
- TLS certificate expiry, hostname mismatch, and wildcard matching analysis.
- TCP JSON export structure and metadata fields.
- Output path resolution and report writing.
- Quiet-mode rendering behavior.
- Local mock TCP, HTTP, TLS, SMTP, FTP, and SMTP STARTTLS services bound only to `127.0.0.1`.
