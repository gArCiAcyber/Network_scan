# HylianScan v0.6 - Release Summary

**Focus:** Style Refactor & Visual Identity Standardization

## Overview
---

### Scope of Version 0.6
This update was entirely dedicated to the aesthetic refactoring and standardization of the scanner's visual identity, introducing the Hylian/Triforce-inspired color palette across all display modules and reports.

---

## Style Refactor & Visual Enhancements

### 1. Main Banner and Footer (CLI Aesthetics)
* **Visual Upgrade:** The main banner received a text restructuring and visual scaling to highlight the project's signature: `[ HYLIANSCAN v0.6 - BY CYLINK ]`.
* **Triforce-Inspired Colors:** Implementation of a customized color palette applied directly to the footer and banner boundaries to create a solid identity right at tool initialization.

### 2. Reporting Module (Subdomain Report)
* **Hylian Color Scheme:** The visual output rendering the subdomain report was completely redesigned.
* **Readability:** Replaced standard terminal colors with a clean, hierarchical color scheme, facilitating rapid visual triage of targets and results directly in the CLI.

### 3. Core Code Architecture (Paint Helper)
* **Color Centralization:** To avoid duplicate code and scattered ANSI strings throughout the project, the color palettes were unified within the `core/` directory.
* **Paint Helper Implementation:** Creation/adjustment of a styling helper function to centralize terminal color rendering, ensuring that any future module rigorously follows the same aesthetic standard without additional effort.

---

## 📂 Modified Files in Scope

* `hylianscan.py` -> Update to the main script core, main banner logic, and subdomain report styling.
* `core/banner.py` -> Adjustments to text positioning, font/color changes for the header, and decorative footer.
