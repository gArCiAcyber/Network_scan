# HylianScan v0.5 Summary

## Overview

Version `v0.5` introduces a second reconnaissance mode to HylianScan: modular subdomain enumeration.

The project now supports two separated operational paths:

- TCP port scanning with `-p / --ports` or `--top-ports`.
- Subdomain brute-force enumeration with `-w / --wordlist`.

The CLI remains centered around the required positional `target` argument, but the selected flags now determine the active mode.

---

## 1. Subdomain Enumeration Module

### What Changed

A new module was added:

```text
modules/subdomain.py
```

This module implements threaded subdomain brute-force enumeration using native Python libraries only:

- `socket` for DNS resolution;
- `threading` for concurrency;
- `queue` for distributing wordlist candidates safely between workers;
- `dataclasses` for structured results.

### Why It Matters

Subdomain enumeration is a core reconnaissance step in offensive security workflows. Keeping it in its own module prevents the TCP scanner from becoming a mixed-purpose monolith.

### Architectural Impact

The architecture now separates scan engines by responsibility:

```text
modules/
├── subdomain.py
├── target.py
└── tcp_scanner.py
```

`tcp_scanner.py` remains responsible for TCP port scanning.

`subdomain.py` is responsible for DNS brute-force enumeration.

`hylianscan.py` only orchestrates which mode should run.

---

## 2. CLI Flag Refactor

### Timeout Flag Change

The timeout flag changed from lowercase `-t` to uppercase `-T`:

```bash
-T / --timeout
```

This prevents a conflict with the new thread-count flag.

### New Threads Flag

A new concurrency flag was added:

```bash
-t / --threads
```

Default:

```text
10
```

This flag controls worker concurrency for supported scan modes.

### New Wordlist Flag

A new wordlist flag was added:

```bash
-w / --wordlist
```

When present, HylianScan uses the target as a base domain and runs subdomain enumeration mode.

---

## 3. Mode Selection Logic

### Subdomain Mode

Command pattern:

```bash
python3 hylianscan.py example.com -w wordlists/subdomains.txt
```

Behavior:

- `example.com` is treated as the base domain.
- each wordlist entry is converted into a candidate subdomain;
- candidates are resolved with DNS;
- valid hostnames are printed immediately;
- a final subdomain panel is rendered.

### TCP Port Scan Mode

Command pattern:

```bash
python3 hylianscan.py 192.168.0.10 -p 80,443
```

or:

```bash
python3 hylianscan.py scanme.nmap.org --top-ports 400
```

Behavior:

- target resolution remains handled by `modules/target.py`;
- TCP scanning remains handled by `modules/tcp_scanner.py`;
- dynamic progress and open-port callbacks remain functional.

### Full TCP Range Shortcut

A new shortcut was added:

```bash
-p -
```

This expands to:

```text
1-65535
```

Example:

```bash
python3 hylianscan.py 192.168.0.10 -p - -t 100 -T 1.0
```

---

## 4. Mode Safety

HylianScan now blocks ambiguous combinations such as:

```bash
python3 hylianscan.py example.com -w subs.txt -p 80,443
```

This is rejected because `--wordlist` activates subdomain mode while `-p` activates TCP mode.

The tool requires a clean mode decision:

- use `-w` for subdomain enumeration;
- use `-p` or `--top-ports` for TCP port scanning.

---

## 5. Output Support

The existing `-o / --output` behavior remains available.

It now supports both:

- TCP scan final panels;
- subdomain enumeration final panels.

Output files are still written inside the project `output/` directory.

---

## 6. Example Commands

### Subdomain enumeration

```bash
python3 hylianscan.py example.com -w wordlists/subdomains.txt -t 20
```

### TCP scan with custom ports

```bash
python3 hylianscan.py scanme.nmap.org -p 80,443 -T 1.5 -t 20
```

### TCP scan with top ports

```bash
python3 hylianscan.py example.com --top-ports 400 -t 50
```

### TCP scan over all ports

```bash
python3 hylianscan.py 192.168.0.10 -p - -T 1.0 -t 100
```

### Save subdomain results

```bash
python3 hylianscan.py example.com -w subs.txt -o subdomains.txt
```

---

## Summary

HylianScan v0.5 expands the project from a TCP scanner into a modular reconnaissance framework.

The main architectural improvement is mode separation: subdomain enumeration and TCP scanning are independent engines, while the CLI simply routes execution based on user intent.
