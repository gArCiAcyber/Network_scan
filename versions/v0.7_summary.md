# HylianScan v0.7 - Release Summary

**Focus:** Passive Subfinder Streaming, Terminal Hygiene, and File-Based Result Management

## Overview

Version 0.7 upgrades passive subdomain discovery from a simple blocking wrapper into a cleaner operational workflow. The scanner now streams Subfinder telemetry into the dynamic terminal line, avoids dumping large result sets into the terminal, and always stores passive findings in a text file.

## Completed Changes

### 1. Subfinder Streaming Execution

- Replaced blocking `subprocess.run()` with `subprocess.Popen()`.
- Read Subfinder `stdout` for discovered subdomains.
- Read Subfinder `stderr` for real-time telemetry.
- Passed telemetry back to the CLI through a callback instead of rendering inside the module.

### 2. Terminal Output Control

- Removed terminal flooding from passive subdomain mode.
- Stopped printing each discovered subdomain directly to the screen.
- Routed active Subfinder telemetry through `write_dynamic_line()`.
- Replaced yellow/orange subdomain status output with `ALERT_RED`.

### 3. Mandatory File Output

- Passive discovery now always writes results to disk.
- Default path without `-o`: `output/hylianscan_subdomains.txt`.
- Directory mode with `-o <directory>`: `<directory>/subdomains.txt`.
- The terminal prints only a final summary containing the count and saved file path.

### 4. Legacy Code Removal

- Removed old multithreaded/socket brute-force subdomain code.
- Removed stale data classes, queues, DNS wildcard logic, and wordlist parsing from `modules/subdomain.py`.
- Kept the module focused on passive Subfinder integration.
