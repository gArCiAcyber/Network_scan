# HylianScan v0.4 Summary

## Overview

Version `v0.4` improves HylianScan in three key areas:

- native Linux terminal input handling with GNU readline;
- clickable web URL generation with ANSI Cyan highlighting;
- active probing banner grabbing for web services using lightweight HTTP `HEAD` requests.

Additionally, this development cycle begins preparing the internal architecture for the future `v0.5` transition by introducing the first CLI argument parsing structures using Python `argparse`.

These changes preserve the current architecture by keeping terminal behavior, visual rendering, target handling, and TCP scanning responsibilities separated.

---

## 1. GNU Readline Terminal Input Fix

### What Changed

The main CLI entrypoint now enables native GNU readline support for the interactive target prompt.

Conceptually:

```python
import readline

readline.parse_and_bind("set editing-mode emacs")
Why This Matters

This fixes Linux terminal issues where arrow keys or backspace could print raw ANSI escape sequences such as:

^[[A
^[[B

The terminal input now behaves like a native shell prompt with proper line-editing support.

2. Clickable URL Generation for Web Services
What Changed

The TCP scanner now automatically generates clickable URLs when common web ports are detected.

Examples:

http://target-ip
https://target-ip
http://target-ip:8080

These URLs are dynamically colorized using ANSI Cyan:

CYAN = "\033[36m"
RESET = "\033[0m"
Why This Matters

This improves reconnaissance workflow speed by allowing direct interaction with discovered web services directly from supported terminals.

3. Active HTTP Banner Probing
What Changed

Banner grabbing logic was upgraded from passive-only receiving to active probing for web services.

The scanner now sends lightweight HTTP requests after establishing TCP connections:

HEAD / HTTP/1.1
Host: target
Why This Matters

Previously, many HTTP/HTTPS servers returned no banner because they waited for client interaction first.

The active probing system now forces compliant web servers to respond with metadata such as:

Server: nginx
Server: Apache

This significantly improves banner collection reliability during reconnaissance.

4. Initial CLI Argument Support (v0.5 Preparation)
Development Status

This is not yet considered the official v0.5 release.

Instead, these changes represent the first architectural preparations for the future CLI-driven workflow planned for v0.5.

What Changed

The project now includes the first implementation of command-line argument parsing using Python argparse.

The previous manual target input flow (read_target()) was removed and replaced with structured CLI parsing.

New functions introduced:

parse_arguments()
parse_ports_list()
Supported Usage

Examples:

python hylianscan.py 192.168.0.10
python hylianscan.py scanme.nmap.org -p 80,443
python hylianscan.py target.com --top-ports 400
Why This Matters

This transition prepares HylianScan for future offensive security tooling standards by enabling:

faster automation;
reproducible scans;
shell scripting integration;
cleaner terminal workflows.

The full CLI ecosystem and advanced automation features remain planned for the future v0.5 milestone.
