# Hylianscan v1.0 Release Checklist

This checklist tracks the final validation work required before tagging Hylianscan v1.0.

## Release Goal

Ship Hylianscan v1.0 as a stable, source-installable, standard-library-first reconnaissance tool for authorized TCP scanning, passive discovery orchestration, and structured TXT/JSON reporting.

## Pre-Release Assumptions

- The release is built from the final `main` branch.
- Public documentation matches the current CLI behavior.
- Demo commands use authorized targets, localhost, or clearly scoped lab targets.
- Passive discovery providers are optional external tools and are documented as separate installs.
- Generated output files are not committed.

## Git State

- [ ] `git status --short` is clean before tagging.
- [ ] The local branch is up to date with `origin/main`.
- [ ] No private local workspace material is staged.
- [ ] No generated `output/` report files are staged.
- [ ] No temporary GIF workshop files are staged.

## Required Validation

Run from the repository root:

```bash
python hylianscan.py --version
python hylianscan.py --help
python -m compileall -q hylianscan.py core modules tests
python -m unittest discover -s tests -p "test_*.py" -v
python scripts/validate_release.py
git diff --check
```

## Packaging Validation

- [ ] `python -m pip install --dry-run .` succeeds.
- [ ] `pipx install .` succeeds in a clean environment.
- [ ] `hylianscan --help` works after `pipx install .`.
- [ ] `pipx install git+https://github.com/gArCiAcyber/Network_scan.git` works after the final release push.

## CLI Smoke Tests

Use safe localhost or explicitly authorized lab targets only.

```bash
python hylianscan.py -u 127.0.0.1 -p 1 --quiet
python hylianscan.py 127.0.0.1 -p 1 --quiet --json-output
python hylianscan.py -u scanme.nmap.org -p 22,80,443 --max-rate 3
python hylianscan.py -u scanme.nmap.org --port-profile web -mc 200,301-304
python hylianscan.py --nmap-xml nmap-results.xml
python hylianscan.py --nmap-xml nmap-results.xml -o --json-output
```

- [ ] `--nmap-xml` imports a single-host sample XML fixture without starting a live scan.

## Passive Discovery Checks

- [ ] `--subfinder` behavior is documented and works when Subfinder is installed.
- [ ] `--amass` behavior is documented and works when Amass is installed.
- [ ] `--subfinder-path` and `--amass-path` errors are clear for invalid paths.
- [ ] Passive TXT output writes `subdomains.txt` in the expected workspace.
- [ ] Passive JSON output writes `subdomains.json` in the expected workspace.

## Output Checks

- [ ] Default TCP TXT output uses `output/<target>/<timestamp>/tcp_report.txt`.
- [ ] Default TCP JSON output uses `output/<target>/<timestamp>/tcp_results.json`.
- [ ] Passive output uses `subdomains.txt` and `subdomains.json`.
- [ ] Nmap XML import output uses `nmap_import_report.txt` and `nmap_import_results.json`.
- [ ] Saved TXT reports do not contain ANSI escape sequences.
- [ ] JSON reports include scan metadata, probe metadata, HTTP metadata, TLS metadata, and filters when applicable.

## Documentation Checks

- [ ] README installation commands are current.
- [ ] README examples parse correctly.
- [ ] README does not mention removed flags such as `--subdomains` or `-w / --wordlist`.
- [ ] README project layout excludes private local directories.
- [ ] `docs/TODO.md` reflects the final v1.0 status.
- [ ] Version summaries remain historical and are not used for unrelated future work.

## GitHub Actions

- [ ] `tests.yml` passes on the final branch.
- [ ] `pipx-install.yml` passes on the final branch.
- [ ] The README test badge points to the correct workflow.

## Go / No-Go

- [ ] All required validation commands pass.
- [ ] GitHub Actions pass.
- [ ] README and CLI help agree.
- [ ] Packaging checks pass.
- [ ] No private or generated files are staged.
- [ ] Tag `v1.0.0` only after the checks above are complete.
